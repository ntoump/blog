In this blogpost, I will present a short review of my attempts to tackle the famous Titanic Dataset machine learning problem.

Its goal is, essentially, to predict on a test dataset of 418 people for each person/entry who lives (portrayed as 1) and who dies (as 0) in the tragic incident. The predictions are made on the basis of the passengers' attributes, such as their gender, their age, and the price of their ticket.

Despite this being my first Kaggle competition, I did achieve a decent final ranking of top 5%. Of course, I have strong reasons to believe that with more time and effort, I would easily slide in the top 1%.

Data Preprocessing

Initially, I started with a standard convention approach: keep all columns that are seemingly different and possibly useful, and classify them as Categorical or Continuous.

The Continuous variables are inputted directly in the model, while the Categorical ones first go through a modification process, which assigns to each level in their domain a unique, albeit inherently meaningless, numerical value.

This helps in differentiating between different values (levels) of Categorical variables, as the models learns to consider a specific arithmetic sequence as the identification factor for each, thereby it can detect relations in the data (e.g. more 0's than 1's - regarding a specific column - tend to survive) without knowing what the values mean (in this example, that women tend to survive more than men - in the category of gender ).

Input of the Training Dataset

From here on out, the training dataset will be represented as df, as it has been inputted as a Pandas DataFrame.

The same goes for the testing dataset, where t_df stands instead.

sb.heatmap(df.isnull())

Seeing that there are many null values in the Cabin column, I decided to drop it from the DataFrame.

Also, I interpolated the null values of the Age column. This is a rather common occurence in Tabular data handling; the default is to either set the mean of the column (which wouldn't help in this case) or to set a specific numerical value for all N/A values (which is preferred here).

df['Age'] = df['Age'].interpolate()
df = df.drop('Cabin',axis=1)
df = df.dropna()

In continuation, I:

  • define the Tabular Procs (as fastai calls the transforms that take place on Tabular data),
  • set the dependent column ("Survived" , where, as I mentioned above, 1 means "true" and 0 means "false),
  • seperate the Continuous from the Categorical columns,
  • remove the Name and the PassengerId columns, since their values are not (contrary to some weird data indications) related to the predictions,
  • split the training and validation set (with the ratio being 0.75-0.25),
  • build the TabularPandas object instance, passing in all the above as its parameters.
procs=[Categorify,FillMissing,Normalize]

Input of the Test Dataset

Seeing that the test dataset is structurally the same as the training dataset, I must apply the same transforms and data preprocessing in order to avoid rather annoying errors that can (and more often than not, do (!)) occur during inference, due to asymmetrical dataset structures.

t_df = pd.read_csv(path/'test.csv')
t_df = t_df.drop('Cabin',axis=1)
t_df['Age']=t_df['Age'].interpolate()
t_df=t_df.fillna(0.0)
too = TabularPandas(t_df,procs, cat, cont, y_block=CategoryBlock)

I should mention that, instead of just dropping whatever rows do not have values, instead I call t df.fillna(0.0) to fill every N/A value with 0.0. This must be done, because since the entries are in a specific order, their corresponding predictions must also follow that same order.

Testing

As I am performing regression, I must interpret the results of the model in a way that makes sense for the labels of this dataset (which are exclusively 0 and 1). Therefore, I created this very simple function that inputs some array (or even Tensor (!)) of predictions, distinguishes between correct and wrong estimations, and returns the accuracy of the model on the validation data.

def acc(xs):
    correct, total = 0, 0
    for pr,lab in zip(xs, y):
        if float(pr)>=0.5: pr = 1.0
        else: pr = 0.0
        if int(pr) == lab: correct += 1
        total += 1
    return float(correct/total)

accuracy = acc(m.predict(valid_xs), valid_y)

Model Overview

The model has a very decent accuracy on the validation data, considering the effort that led to its creation. As it is, however, but a baseline model, I will try to improve on that initial score of ~82.02% before submitting on Kaggle.
